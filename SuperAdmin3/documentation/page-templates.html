Page Templates

Super Admin uses 4 different page templates to help you get started with your project. Each template has a different layout and is designed to be used in different scenarios based on the content of the page.

Two Column - Sidebar and Body

The two column template is the most common layout used in Super Admin. It has a sidebar on the left and a body on the right. The sidebar is used for navigation and the body is used for the main content of the page.

e-commerce.html
crm.html
faq.html
invoice.html
<!DOCTYPE html>
<html lang="en" data-page="[kebab-cased Page Title]">
	<!-- Head -->
	<module href="./src/html/partials/_head.html" locals='{ "pageTitle": "Page Title" }'></module>
	
	<body>
		<!-- Page loader -->
		<module href="./src/html/partials/_page-loader.html"></module>

		<!-- Site header -->
		<module href="./src/html/partials/_header.html"></module>

		<!-- Content header -->
		<header class="content-header">
			<h2 class="m-0 text-body-emphasis fs-6">E-Commerce</h2>
		</header>

		<!-- Content -->
		<div id="content">
			<!-- Site sidebar -->
			<module href="./src/html/partials/_sidebar.html"></module>
			
			<!-- Page content goes here... -->
		</div>

		<!-- Scripts -->
		<module href="./src/html/partials/_script.html"></module>
	</body>
</html>

Three Column - Sidebar, Content Sidebar and Body

The three-column template is employed when you want to showcase a content-level sidebar on the left side of the page. This is particularly handy when you need to present a list of items on the left and the main content on the right.

How it works:

  • In this layout, the default sidebar that contains the menu, is shrunken to make room for the content level secondary sidebar by adding sidebar-min class to the body tag.
  • A helper class content-column is added to the #content div to have the columned layout.
  • A distinctive ID is assigned to the content-level sidebar, known as sidebar-secondary.
  • A button with sidebar-secondary-toggle ID is placed within _content-header.html to toggle the secondary sidebar in mobile and smaller screens.
  • On mobile devices and smaller screens, only the content body is shown, and you can toggle the secondary sidebar by clicking the toggle button mentioned previously.
file-manager.html
messages.html
photos.html
calendar.html
<!DOCTYPE html>
<html lang="en" data-page="[kebab-cased Page Title]">
	<!-- Head -->
	<module href="./src/html/partials/_head.html" locals='{ "pageTitle": "Page Title" }'></module>
	
	<body class="sidebar-min">
		<!-- Page loader -->
		<module href="./src/html/partials/_page-loader.html"></module>

		<!-- Site header -->
		<module href="./src/html/partials/_nav-bar.html"></module>

		<div id="content" class="content-column">
			<!-- Site sidebar -->
			<module href="./src/html/partials/_sidebar.html"></module>

			<!-- Content sidebar -->
			<module href="./src/html/partials/photos/_content-sidebar.html"></module>

			<!-- Main -->
			<div class="content-inner">

				<!-- Content header -->
				<module href="./src/html/partials/photos/_content-header.html"></module>

				<div class="content-body" data-simplebar>
					<!-- Page content goes here... -->
				</div>
			</div>
		</div>

		<!-- Script -->
		<module href="./src/html/partials/_script.html"></module>
	</body>
</html>

<aside class="content-sidebar" id="sidebar-secondary">
    <!-- Close button -->
	<button type="button" class="icon sidebar-close d-xl-none" id="sidebar-secondary-close">
		<i class="ph ph-x pe-none fs-5"></i>
		<span class="visually-hidden">Close</span>
	</button>

	<!-- Body -->
	<div class="content-body" data-simplebar>
		<!-- Content goes here... -->
	</div>
</aside>

<div class="content-header">
	<!-- Secondary sidebar toggle -->
	<module href="../_secondary-sidebar-toggle.html"></module>

    <!-- Title -->
	<h2 class="fs-5 text-body-emphasis m-0">Title</h2>
</div>

Four Column - Sidebar, Content Sidebar, Content List and Body

Like the three column layout, the four column layout is also a combination of the sidebar, content sidebar and body sections. The difference is that the four column layout also includes an additional section for a third sidebar.

How it works:

  • Follow the first 4 steps as described previously in the 3 column layout.
  • Both the list level third sidebar and content body sections are labeled with the data-attribute data-sa-block = { list | body }. This is done to ensure that the layout is responsive and properly displayed on mobile devices and smaller screens.
  • On mobile devices and smaller screens, one column is displayed at a time and links/buttons with the data-sa-toggle = { list | body } attributes are used to toggle between the columns.
contacts.html
mail.html
<!DOCTYPE html>
<html lang="en" data-page="[kebab-cased Page title]">
	<!-- Head -->
	<module href="./src/html/partials/_head.html" locals='{ "pageTitle": "Page title" }'></module>
	
	<body class="sidebar-min">
		<!-- Page loader -->
		<module href="./src/html/partials/_page-loader.html"></module>

		<!-- Site header -->
		<module href="./src/html/partials/_nav-bar.html"></module>
		
		<div id="content" class="content-column">
			<!-- Site sidebar -->
			<module href="./src/html/partials/_sidebar.html"></module>

			<!-- Content sidebar -->
			<module href="./src/html/partials/contacts/_content-sidebar.html"></module>

			<!-- Content list -->
			<module href="./src/html/partials/contacts/_content-list.html"></module>

			<div data-sa-block="body" class="content-inner">
				<!-- Content header -->
				<module href="./src/html/partials/contacts/_content-header.html"></module>
				
				<!-- Body -->
				<div class="content-body" data-simplebar>
					<!-- Page content goes here... -->
				</div>
			</div>
		</div>

		<!-- Script -->
		<module href="./src/html/partials/_script.html"></module>
	</body>
</html>

No Columns - Body only

No columns layout is a simple layout with only the body content. It is useful for pages that don’t need a sidebar or a list.

login.html
coming-soon.html
forgot-password.html
<!DOCTYPE html>
<html lang="en">
	<!-- Head -->
	<module href="./src/html/partials/_head.html" locals='{ "pageTitle": "Login" }'></module>
	
	<body>
		<!-- Page content goes here... -->

		<!-- Scripts -->
		<module href="./src/html/partials/_script.html"></module>
	</body>
</html>
